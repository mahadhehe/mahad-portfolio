Designing Responsive Portfolio Cards with CSS Grid

April 19, 2025

Creating clean, aligned cards for my portfolio was a game-changer. I used CSS Grid to build responsive layouts for my Projects and Blog pages. Hereâ€™s my process.

Why CSS Grid? Grid offers precise control over rows and columns, perfect for card layouts. I set up a container with display: grid; grid-template-columns: repeat(auto-fit, minmax(400px, 1fr)); to ensure cards adapt to screen sizes while staying big and balanced.

Card Design: Each card stacks an image, title, text, and button using flex-direction: column. I added glassmorphism with backdrop-filter: blur(10px) for a sleek look, matching my portfolioâ€™s vibe. Images are capped at max-width: 600px to keep things bold but not overwhelming.

Responsiveness: On mobile, Grid stacks cards vertically with grid-template-columns: 1fr. I tweaked font sizes (e.g., 2rem to 1.5rem) and padding for smaller screens, testing on devices to ensure no overlap.

Challenges: Early layouts had uneven gaps. I fixed this with gap: 2rem and centered cards using place-items: center. Another issue was image scalingâ€”I used object-fit: cover to prevent stretching, inspired by X posts on responsive design.

Tip for Devs: Plan your grid early and test across breakpoints. Use DevTools to catch alignment issues. See my Projects page for the result!
